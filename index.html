Git

Git'as yra populiariausia kodo versijavimo programa.

Git komandos

  • init - sukuria git'o repozitoriją dabartiniame aplankale
  • status - nurodo informaciją apie lokalios git'o repozitorijos statusą
  • add - prideda nurodytus failus prie commit'ui paruoštų failų (stage'ina)
  • rm --cached - išima nurodytus failus iš stage'inimo zonos (unstage'ina)
  • commit -m "žinutė" - sukuria kodo versiją iš stage'intų failų ir prideda žinutę apie pakitimus (žinutė turėtų būti trumpa (ne ilgesnė nei 50 simbolių) ir aiški)
  • restore - grąžina nurodytus failus į buvusio commit'o būseną (panaikina naujus pakitimus padarytus po paskutinio commit'o)
  • push - iš lokalios repozitorijos nusiunčia commit'us į išorinę repozitoriją
  • pull - iš išorinės repozitorijos atsiunčia commit'us į lokalią repozitoriją
  • clone - klonuoja nurodytą repozitoriją iš išorinės repozitorijos
  • log - išveda commit'ų sąrašą
  • checkout commitID|branchName - pareina į nurodytą kodo versiją ARBA į kitą šaką
  • branch naujosSakosPavadinimas - sukuria naują šaką

Branch protection rules

Vienas svarbiausių šakų apsaugos taisyklių yra "Pull Request" (bandymas vienos šakos commit'us sukelti į kitą šaką).
Pull Request dar galima suteikti "Require approvals", kuris reikalauja, kad nurodytas kiekis kitų projekto dalyvių patvirtintų Pull Request'ą.
